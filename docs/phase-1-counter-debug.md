# Phase 1：3C 自动计数调试记录

## 当前结论

`3C` 自动计数没有更新，不是因为会话没启动，而是因为状态识别没有稳定命中：

- 当前会话会停在 `waiting-for-game`
- `completedRuns` 不增长
- `runHistory` 不新增
- 最近识别结果常见为 `unknown`

这说明监控进程在跑，但没有成功把画面稳定判定为：

- `lobby`
- `in-game`

## 已确认根因

第一轮排查已经确认，之前的判定链过于依赖“单次硬阈值”：

- 例如大厅模板可能打到 `0.685`
- 但硬阈值是 `0.700`
- 结果就是“明明很接近，但一次不过就直接算失败”

这会导致两种问题：

1. 分辨率、UI 缩放略有变化时，模板分数会整体偏低
2. 单帧波动会让状态在 `unknown` 和真实状态之间反复跳

## 本轮已落地的修正

本轮已经重写 [run_counter_monitor.py](/E:/暗黑2桌宠/automation/python_runtime/tasks/run_counter_monitor.py)，核心变化有 3 个：

1. 多尺度模板匹配
   - 状态模板不再只按单一尺寸匹配
   - 会在一小组缩放比例里寻找最佳分数

2. 软阈值稳定确认
   - 保留原来的“硬阈值直接命中”
   - 新增“稳定识别阈值”
   - 当某个状态连续两次进入稳定区间时，也会被确认

3. 更清楚的诊断信息
   - 当前命中的模板名
   - 当前分数
   - 当前使用的阈值
   - 是否处于“等待稳定确认”
