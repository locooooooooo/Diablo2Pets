# 暗黑 2 桌宠助手产品重置说明

## 1. 这款应用真正要做什么

这不是一个“桌宠展示器”，也不是一个“功能很多的游戏面板集合”。

它的核心定位应该是：

- 一个常驻桌面的暗黑 2 陪刷助手
- 一个围绕刷图主流程服务的轻量工具
- 一个把你已经有的自动化能力接进统一入口的桌面壳

它首先要解决的，是下面 4 条主线：

1. 陪刷计数
2. 掉落记录
3. 自动化工坊
4. 桌宠常驻陪伴

## 2. 产品主线

### 2.1 陪刷计数

目标：

- 记录今天刷了多少把
- 每把用了多久
- 总共刷了多久
- 默认优先服务 3C 这样的固定路线

理想体验：

1. 打开桌宠
2. 自动待命
3. 识别进入大厅
4. 识别进入游戏
5. 自动开始一把
6. 回大厅自动结算一把
7. 首页直接看到累计把数、当前把耗时、平均耗时、最近时间线

### 2.2 掉落记录

目标：

- 以最低操作成本记录掉落
- 支持截图粘贴
- 支持 OCR 回填
- 支持按天回看

理想体验：

1. 刷到东西截图
2. 回桌宠按 `Ctrl+V`
3. 自动保存截图
4. 自动 OCR
5. 一键保存
6. 今天战报里能立刻看到

### 2.3 自动化工坊

目标：

- 把你已经做好的 3 条自动化能力接入桌宠
- 只保留一条清晰操作路径
- 明确告诉用户当前缺什么

当前优先支持：

- 符文自动合成
- 宝石自动合成
- 共享仓库丢金币

理想体验：

1. 打开工坊
2. 先看当前阻塞项
3. 缺环境就补环境
4. 缺坐标配置就录配置
5. 先试运行
6. 再正式执行

### 2.4 桌宠常驻陪伴

目标：

- 常驻桌面
- 不打扰
- 轻量提示
- 有一点个性，但不能压过主流程

这部分的优先级应该排在核心可用性之后。

## 3. 当前范围应该砍掉什么

为了回到你最初想要的应用，当前阶段应该明确降级这些内容：

- 宠物房间
- 收藏墙
- Atlas 图谱
- 稀有掉落剧情册
- 过多成长系统
- 复杂场景切换

这些内容不是永远不要，而是暂时不应该继续占据首页和第一屏注意力。

## 4. 当前版本应该是什么

现阶段的目标版本应该定义为：

### V1 核心版

只保证以下能力稳定可用：

- 桌宠窗口稳定打开、滚动、切页、收起、悬浮
- 3C 自动计数稳定可用
- 掉落截图记录稳定可用
- 工坊三条任务能完成“预检 -> 录配置 -> 试运行 -> 执行”

### V1.5 体验版

在 V1 稳定后再补：

- 桌宠对白
- 钓鱼待机
- 简单成长反馈
- 更完整的战报统计

### V2 陪伴版

最后再考虑：

- 收藏册
- 房间成长
- 场景演出
- 稀有掉落剧情化

## 5. 页面结构应该重置成什么

### 首页：陪刷

第一屏只保留：

- 当前状态
- 当前主动作
- 当前路线统计
- 一个“展开详情”

### 第二页：战报

第一屏只保留：

- 快速录入
- 最近几条
- 一个“查看详细战报”

### 第三页：工坊

第一屏只保留：

- 当前阻塞项
- 当前主任务
- 当前任务卡
- 一个“展开更多诊断”

### 桌宠壳

默认只保留：

- 头像
- 标题
- 悬浮 / 置顶 / 收起
- 当前一句状态提示

## 6. 成功标准

当这个产品做对时，用户应该能做到：

1. 打开桌宠后，第一眼知道现在该做什么
2. 不看代码、不看日志，也能走完主流程
3. 跑几把 3C 后，首页会自动更新次数和耗时
4. 刷到东西后，能很快记进战报
5. 需要自动化时，工坊不会让人迷路
