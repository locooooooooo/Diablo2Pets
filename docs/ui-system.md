# 暗黑 2 桌宠助手 UI 系统规范

## 1. 文档目的

这份文档定义当前版本的 UI / UX 设计系统基线，用于保证：

- 后续页面扩展时视觉一致
- 面板态和悬浮态属于同一产品
- 新功能接入时不破坏信息层级
- UI 优化不再依赖“凭感觉修改”

---

## 2. 设计方向

当前产品采用的主视觉方向是：

## 赫拉迪姆仪表盘

核心关键词：

- 黄铜
- 羊皮
- 仪轨
- 圣物
- 桌面召唤器

这意味着它不是现代 SaaS 面板，也不是赛博霓虹工具箱。

### 应该给人的感觉

- 常驻桌面但不轻浮
- 有暗黑 2 气质，但不是直接抄游戏 UI
- 看起来像桌面上的炼金术器具，而不是脚本启动器

### 明确避免

- 默认后台管理面板风格
- 过度冷白或纯黑极简
- 紫白渐变类 AI 通用视觉
- 大量装饰但没有信息主次

---

## 3. 页面层级

## 一级层级

一级导航只有三个：

1. `陪刷`
2. `战报`
3. `工坊`

### 层级规则

- `陪刷` 是最高频入口
- `战报` 是刷图收口入口
- `工坊` 是低频高级入口

## 二级层级

### 陪刷

- 当前状态
- 今日战况
- 快捷流转
- 今日战果预览

### 战报

- 今日战报摘要
- 快速记录表单
- 筛选区
- 今日明细

### 工坊

- 统一执行策略
- 三类任务卡
- 高级信息折叠区
- 日志查看区

---

## 4. 视觉 token

## 色彩角色

### 背景

- `--pet-bg-0`
- `--pet-bg-1`
- `--pet-bg-2`

用于主背景、深层容器和悬浮态底色。

### 面板

- `--pet-panel`
- `--pet-panel-soft`
- `--pet-panel-glow`

用于卡片、浮层、工具块和局部高亮。

### 描边

- `--pet-border`
- `--pet-border-strong`

用于卡片边框、激活边框、hover 强化边缘。

### 文本

- `--pet-text-0`
  主标题、主数字、重点信息
- `--pet-text-1`
  次级正文、标签、说明
- `--pet-text-2`
  辅助说明、空态、提示

### 强调色

- `--pet-accent`
  常规强调
- `--pet-accent-hot`
  主 CTA、危险或热操作强调

---

## 5. 字体系统

## 展示字体

展示标题使用 serif 系风格，承接“圣物 / 仪轨 / 档案”气质。

适用对象：

- 页标题
- 大数字
- 卡片主标题

## 正文字体

正文使用可读性更强的中文界面字体。

适用对象：

- 按钮
- 输入框
- 标签
- 列表文本
- 帮助说明

## 字号层级建议

- 页面标题：18-24
- 卡片标题：16-18
- 正文：14-16
- 辅助说明：12-13
- 关键数字：24+

---

## 6. 组件规范

## 按钮

### 主按钮

用于：

- 开始刷图
- 结束刷图
- 立即执行
- 保存掉落
- 展开面板

规则：

- 每个局部区域只保留一个最强主按钮
- 主按钮使用暖色高对比渐变
- 高度不低于 44px

### 次按钮

用于：

- 试运行
- 查看日志
- 查看 profile
- 打开截图

规则：

- 视觉上低于主按钮
- 不能和主按钮抢焦点

### 文本按钮

用于：

- 打开文件
- 轻量跳转
- 次级查看动作

规则：

- 仅用于真正低优操作
- 不可承担关键提交动作

## 卡片

规则：

- 所有主要信息块必须卡片化承载
- 卡片分三层：
  - 主卡
  - 次卡
  - 内嵌辅助块
- 卡片不能只靠颜色区分，还要靠留白和标题区分

## 输入区

规则：

- 所有输入必须有明确 label
- 复杂输入旁边必须有 helper text
- 数值输入优先靠近对应动作按钮
- 输入与执行按钮之间距离不能过大

## Toast

规则：

- 成功和失败必须有明显区分
- 文案优先说用户结果，不优先说技术动作
- 3-5 秒自动消失

---

## 7. 状态设计

## 全局状态

- Booting
- Idle
- Hunting
- Drop Highlight
- Task Running
- Task Success
- Task Error

## 页面状态

每个页面必须至少具备：

- 默认态
- 空态
- 加载 / 忙碌态
- 失败态

## 交互反馈规则

- 成功：优先告诉用户“发生了什么”
- 失败：优先告诉用户“接下来怎么办”
- 高亮掉落：允许更情绪化、更桌宠化

---

## 8. 悬浮态规范

悬浮态不是主面板缩放版。

### 目标

- 更像桌宠
- 更轻
- 更快
- 更聚焦

### 必备元素

- 宠物主体
- 状态气泡
- 展开按钮
- 战报入口
- 工坊入口
- 置顶
- 收起

### 禁止事项

- 不要在悬浮态塞完整表单
- 不要塞长文本日志
- 不要塞多层配置

---

## 9. 可访问性与可用性底线

## 必须满足

- 主要可点击区域不低于 44px
- 焦点态可见
- 不能只靠颜色表达状态
- 主要文本与背景对比度足够
- 动效支持 reduced motion 降级

## 推荐满足

- 键盘可切页
- 键盘可触发主操作
- 图标按钮具备文本或 aria 语义

---

## 10. 文案语气

产品文案建议保持：

- 简洁
- 鼓励
- 有陪伴感
- 不要太“脚本工具”

### 示例

更推荐：

- `已经开始记录本次刷图`
- `掉落记录已经保存`
- `试运行已经完成`

不推荐：

- `任务进程已启动`
- `命令执行成功`
- `stdout 已更新`

---

## 11. 后续扩展规则

后续新增页面或模块时，必须先回答：

1. 它属于陪刷、战报还是工坊哪一侧。
2. 它是高频动作还是低频配置。
3. 它该进入面板态还是悬浮态。
4. 它是否遵守现有按钮、卡片、状态和反馈规则。

如果这四个问题回答不清，就不应该直接开始写 UI。
