# 暗黑2桌宠助手 - 技术实施与验收计划

基于《UI/UX优化与大规模玩家工具箱演进方案》，我们将优化目标拆解为可落地的代码执行步骤。实施将分为多个阶段，每个阶段包含明确的执行目标、涉及文件、验证标准与验收测试。

---

## 阶段一：核心主路径极简化 - 首页（陪刷）重构

### 1. 目标
化繁为简，首屏只保留“当前刷图状态”和极具视觉引导的“开始/结束”主按钮。将复杂的统计图表、战果预览默认折叠到二级“今日详细”面板中。

### 2. 涉及文件
- `src/components/CounterPanel.tsx`

### 3. 执行方案
- 引入一个折叠状态 `showDetails`，默认值为 `false`。
- 将原本直接展示的 `TodayStats` (今日战报/效率) 和 `Active Run` (当前状态) 的次要信息进行分离。
- 顶部突出核心的“开始刷图”/“结束刷图”大按钮。
- 增加一个“展开/收起今日详细战报”的切换按钮。

### 4. 验收与测试
- **静态检查**：运行 `npm run build:renderer` 无类型错误。
- **UI 验收**：首页高度大幅缩减，首屏无图表干扰；点击“展开详情”后平滑显示统计数据。

---

## 阶段二：核心主路径极简化 - 战报（掉落）体验优化

### 1. 目标
强化“Ctrl+V 直接粘贴”的心智引导。首屏仅展示最近的掉落快照，弱化繁杂的表单填写（将其作为次要展开项或悬浮态）。

### 2. 涉及文件
- `src/components/DropPanel.tsx`

### 3. 执行方案
- 顶部增加醒目的“请在此页面直接按下 Ctrl+V 粘贴游戏截图”占位提示框。
- 隐藏默认完全展开的手动录入表单，将其改为“手动录入”按钮触发的折叠面板。
- 列表区域仅渲染最近 3-5 条记录，并提供“查看全部”的入口（当前可仅作数量限制）。

### 4. 验收与测试
- **静态检查**：运行 `npm run build:renderer` 无类型错误。
- **UI 验收**：进入战报页第一眼看到的是粘贴引导，而非复杂的输入框。

---

## 阶段三：工坊（自动化）向导化与文案降噪 (已完成)

### 1. 目标
重构为“向导式（Wizard）”操作流，并把极客术语翻译为玩家易懂的“人话”。

### 2. 涉及文件
- `src/components/AutomationPanel.tsx`

### 3. 执行方案
- **术语替换**：
  - `Profile` -> `坐标与范围配置`
  - `Runtime` -> `运行环境`
  - `OCR引擎` -> `文字识别组件`
- **向导化**：当前如果是“未配置环境”状态，隐藏下方的“试运行/执行”按钮，只给出“第一步：修复/安装运行环境”的引导。只有前置条件满足后，才展示下一步。

### 4. 验收与测试
- **静态检查**：代码无错误。 (已通过)
- **UI 验收**：工坊界面术语友好，小白玩家能明确知道当前卡在哪一步（环境没装好，还是坐标没配置）。(已通过)

## 阶段四：桌宠态与工具态解耦 (已完成)

### 1. 目标
挂机时隐藏复杂的控制面板，仅保留极简的透明桌宠悬浮窗。

### 2. 涉及文件
- `src/App.tsx`
- `src/components/PetShell.tsx`

### 3. 执行方案
- 复用现有的 `windowMode` (`panel` | `floating`) 状态，因为主进程已经支持这两种模式的窗口尺寸与透明度解耦。
- 在 `PetShell` 的顶部 drag-strip 区域增加双击事件，触发切换到悬浮态（`floating`）。在悬浮态双击气泡已支持切回面板态。
