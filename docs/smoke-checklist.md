# 暗黑 2 桌宠助手手工回归清单

## 1. 使用说明

这份清单用于每次以下场景前后做 smoke test：

- UI 重构
- 打包前
- 打包后
- 发布前
- 合并高风险功能分支后

建议执行顺序：

1. `dev` 模式跑一遍
2. `build` 模式确认一遍
3. 打包版 exe 再跑一遍关键路径

---

## 2. 环境准备

### 开发模式

- `npm.cmd install`
- `npm.cmd run dev`

### 构建模式

- `npm.cmd run build`

### 打包模式

- `npm.cmd run dist:win`

### 运行前确认

- 没有旧的 `Diablo2Pets.exe` 进程占用打包目录
- 本机 Python 可用，或者明确当前要验证的是“不含自动化”的 UI 壳

---

## 3. 启动与壳层

## 3.1 启动成功

- 桌宠窗口成功显示
- 无白屏
- 无只有标题栏没有内容的问题
- 无 preload / renderer 加载失败

## 3.2 无边框与透明

- 窗口没有系统标题栏
- 窗口背景透明
- 圆角和主视觉正常显示

## 3.3 托盘与收起

- 点击收起后隐藏到托盘
- 点击托盘图标能重新显示
- 托盘菜单可打开

## 3.4 全局热键

- `Alt+Shift+D` 可隐藏窗口
- `Alt+Shift+D` 可再次呼出窗口

---

## 4. 模式切换

## 4.1 面板态

- 默认能进入面板态
- 三个一级导航可切换
- 面板态尺寸和布局正常

## 4.2 悬浮态

- 点击悬浮按钮后切换成功
- 悬浮态尺寸明显小于面板态
- 悬浮态可点击“展开”
- 悬浮态可点击“掉落”
- 悬浮态可点击“工坊”

## 4.3 置顶状态

- 面板态可切换置顶
- 悬浮态可切换置顶
- 托盘菜单切换置顶后，界面状态同步

---

## 5. 陪刷主流程

## 5.1 开始刷图

- 在陪刷页输入地图名
- 点击开始后进入进行中状态
- 计时文本开始更新

## 5.2 结束刷图

- 点击结束后生成一条 runHistory
- 今日次数增加
- 今日总耗时增加
- 平均单次刷新

## 5.3 首页回看

- 最近刷图列表显示最新一条
- 今日掉落预览正常展示
- 首页可以跳到战报
- 首页可以跳到工坊

---

## 6. 战报主流程

## 6.1 粘贴截图

- 切到战报页面
- `Ctrl+V` 粘贴图片成功
- 页面出现截图预览

## 6.2 OCR

- OCR 成功时显示建议物品名
- OCR 成功时显示建议备注
- OCR 结果区显示引擎和识别文本

## 6.3 保存掉落

- 可保存掉落记录
- 保存后列表新增一条记录
- 截图路径可打开

## 6.4 异常兜底

- OCR 失败时仍可手动填写并保存
- 没有截图时不会误保存空记录
- 筛选不会导致页面报错

---

## 7. 工坊主流程

## 7.1 符文

- 输入数量可试运行
- 试运行后可查看日志
- 正式执行按钮可点击
- profile 可查看
- profile 可录制
- 旧配置可导入

## 7.2 宝石

- 矩阵模式可试运行
- 截图识别模式可粘贴图片
- 图片路径模式可打开路径
- 日志查看正常

## 7.3 金币

- 输入金额与等级可试运行
- 日志查看正常
- profile 查看与录制正常

## 7.4 共性检查

- 任务运行时 busy 态正确
- 成功和失败都有反馈
- 日志不会打开空白弹层

---

## 8. 设置与系统能力

## 8.1 开机自启

- 开关可切换
- 状态会回写到界面

## 8.2 系统通知

- 开关可切换
- 刷图结束时可触发通知
- 保存掉落时可触发通知

## 8.3 本地目录

- 截图目录可打开
- 自动化日志目录可打开
- `data.json` 能正常更新

---

## 9. 打包版专项检查

打包版必须额外确认：

- 启动无白屏
- preload 正常加载
- 资源路径无 `file://` 兼容问题
- 图标正确显示
- 托盘图标正常
- 悬浮态正常切换
- 主界面文本不缺失

---

## 10. 通过标准

一次 smoke 通过，至少满足：

- 启动 / 模式切换 / 陪刷 / 战报 / 工坊 五大块都能完成一遍
- 无阻断性白屏、崩溃、无响应
- 打包版至少能完成启动、导航、悬浮切换和基础记录流程

如果以下任一情况出现，则本次 smoke 视为失败：

- 打不开
- 只显示空壳
- 无法切页
- 无法记录刷图
- 无法记录掉落
- 工坊入口不可用
