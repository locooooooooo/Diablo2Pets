# 暗黑 2 桌宠定位与布局重构

## 1. 重新定位

当前版本已经具备计数、掉落记录、OCR、符文/宝石/金币自动化、托盘和全局热键等完整能力，下一步不再是继续堆功能，而是让它更像“桌宠”而不是“脚本面板”。

新的产品定位：

- 它首先是一个陪刷伙伴，常驻桌面，帮你维持刷图节奏和战果感知。
- 它其次是一个战利品账本，负责把截图、OCR、备注和每日掉落沉淀下来。
- 它最后才是一个赫拉迪姆工坊，把高阶自动化和 profile 工具收进第二层。

一句话定义：

> 一个围绕暗黑 2 刷图节奏设计的桌面陪刷助手，而不是把多个脚本塞进同一个壳。

## 2. 功能主次

### P0：桌宠首页

这是用户最常看到的层，应该承担：

- 当前是否在刷图
- 本次刷图耗时
- 今日次数、总耗时、平均单次
- 一键开始/结束本次刷图
- 快捷跳转到掉落页和工坊
- 今日战果预览

### P1：战利品账本

这是刷图收口层，负责：

- `Ctrl+V` 粘贴截图
- OCR 自动识别
- 掉落条目保存
- 今日掉落回看

### P1：赫拉迪姆工坊

这是高级操作层，负责：

- 低级符文自动合成
- 宝石自动合成
- 共享仓库丢金币
- profile 录制
- 旧配置导入
- 执行日志查看

### P2：系统级陪伴能力

这些能力不抢主界面，但决定桌宠体验：

- 托盘常驻
- 全局热键
- 开机自启
- 通知提醒
- 后续的待机动作、气泡、事件提示

## 3. 布局原则

### 原则一：桌宠先表达状态，再表达工具

用户打开窗口时，第一眼应该看到“现在刷到哪里、今天刷了多少、下一步点哪里”，而不是三大工具表单。

### 原则二：高频动作前置，低频配置后置

开始/结束刷图、记录掉落是高频动作，应该放首页。
profile 录制、导入旧配置、看日志是联调动作，应该放在工坊深处。

### 原则三：让桌面状态有陪伴感

桌宠头部需要承担角色感：

- 当前状态
- 今日进度
- 功能身份标签
- 托盘与快捷键提示

## 4. 当前版推荐信息架构

### 顶部：桌宠身份区

- 桌宠名称
- 当前状态
- 今日概览
- 置顶 / 自启 / 通知开关
- 功能身份标签：`陪刷首页 / 战利品账本 / 赫拉迪姆工坊`

### 一级导航

- `陪刷首页`
- `战利品账本`
- `赫拉迪姆工坊`

### 陪刷首页内部

分成 4 块：

1. 当前狩猎状态
2. 今日战况
3. 快捷流转入口
4. 今日战果预览

### 战利品账本内部

分成 3 块：

1. 截图粘贴与 OCR 预览
2. 掉落编辑表单
3. 今日掉落列表

### 赫拉迪姆工坊内部

分成 4 层：

1. 工坊说明
2. 三类任务卡
3. profile 工具区
4. 日志查看器

## 5. 这次落地后的取舍

这次重排不改变底层能力，重点是主次重排：

- 保留现有 1/2/3 自动化 runtime
- 保留掉落 OCR 流程
- 不把自动化工具塞回首页
- 首页改成“状态台 + 快捷流转 + 战果预览”

也就是说，当前阶段不追求“桌宠会动很多”，而是先保证：

- 桌面上像一个陪刷助手
- 打开后顺手
- 现有能力有层次

## 6. 后续可以继续推进的方向

### 方向 A：更像宠物

- 待机动作
- 情绪切换
- 掉落高亮时的对话气泡

### 方向 B：更像战报助手

- 每周 / 每月统计
- 地图维度收益趋势
- 稀有掉落高亮卡片

### 方向 C：更像工坊控制台

- 任务模板
- 最近一次输入记忆
- 自动化执行队列

## 7. 结论

后续这只桌宠的方向应该稳定为：

**桌面上的暗黑 2 陪刷状态台，掉落账本和自动化工坊作为两翼。**

这样既不会丢掉你已经做完的本地自动化能力，也能让整个产品开始长出真正的桌宠气质。
