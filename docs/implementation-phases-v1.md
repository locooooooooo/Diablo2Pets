# 暗黑 2 桌宠助手分阶段实施计划

## Phase 0：重置基线

目标：

- 用文档重新统一方向
- 停止继续扩散功能
- 明确当前版本只服务 V1 核心版

交付：

- 产品重置说明
- 现状与差距文档
- 分阶段实施计划

验收：

- 后续开发按这 3 份文档执行

## Phase 1：修通 3C 自动计数

这是当前最高优先级。

### 目标

让用户跑 3C 时，首页统计会真实更新：

- 总把数
- 每把耗时
- 总耗时

### 子步骤

1. 加强状态识别观测
2. 固化 3C 首套模板
3. 放宽或分层模板阈值
4. 增加识别失败时的人话提示
5. 用真实游戏连续跑几把做验收

### 交付

- 可观测的识别结果
- 稳定的 3C 模板
- 自动计数闭环

### 验收

1. 开启全自动
2. 连续跑 3 把 3C
3. `runHistory` 新增 3 条
4. 首页正确显示累计把数和总耗时

## Phase 2：首页重构成任务仪表盘

### 目标

首页不再是“复杂信息墙”，而是“现在做什么”仪表盘。

### 子步骤

1. 只保留当前状态、当前主动作、当前统计、展开详情
2. 把成长、收藏、房间全部收进二级层
3. 把文案压短成一句人话

### 验收

- 不滚动也能完成“开始 / 停止 / 看统计”主动作

## Phase 3：工坊闭环重构

### 目标

工坊第一屏只回答一个问题：

`你现在缺什么，下一步点哪里`

### 子步骤

1. 环境区重构为“当前是否可用”
2. 坐标配置录制收成单一路径
3. 任务卡只展示当前任务
4. 详细日志和高级诊断全部折叠

### 验收

- 第一次进入工坊的人，能在 10 秒内知道下一步点哪里

## Phase 4：战报闭环重构

### 目标

让掉落记录变成真正的高频动作，而不是后台功能。

### 子步骤

1. 第一屏只保留快速录入
2. OCR 结果更明确
3. 保存反馈更明确
4. 最近几条与详细战报分层

### 验收

- 从截图到保存一条掉落，不超过 3 次点击

## Phase 5：文字与错误系统统一

### 目标

把“看不懂”这个问题一次性收掉。

### 子步骤

1. 清理乱码
2. 统一状态词
3. 统一错误提示
4. 统一引导文案

### 验收

- 不看日志也能理解大部分状态

## Phase 6：桌宠轻陪伴回填

### 目标

在不打扰功能主线的前提下，保留你想要的桌宠感。

### 子步骤

1. 保留 Q 版迪亚波罗待机
2. 保留轻量对白和钓鱼动画
3. 把复杂成长系统收进二级入口

### 验收

- 有陪伴感，但首页和工坊不再被宠物演出抢焦点

## Phase 7：发布前清理与验收

### 目标

把当前工程打磨到可以长期自己用。

### 子步骤

1. 清理残留调试文件
2. 检查打包链
3. 走一遍完整主流程
4. 修最后一批白屏 / 文案 / 滚动问题

### 验收

按以下路径全流程通过：

1. 打开桌宠
2. 开启 3C 全自动
3. 跑几把 3C
4. 首页统计更新
5. 记一条掉落
6. 进入工坊试运行任一任务

## 建议执行顺序

严格按这个顺序推进：

1. Phase 0
2. Phase 1
3. Phase 2
4. Phase 3
5. Phase 4
6. Phase 5
7. Phase 6
8. Phase 7
