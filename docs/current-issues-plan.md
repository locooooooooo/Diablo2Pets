# 当前问题与分步实施计划

## 当前核心问题

### 1. 信息架构过重
- 首页、桌宠壳、战报、工坊都承载了过多信息。
- 同一类信息重复出现，用户第一眼看不到“现在该做什么”。
- 桌宠成长、收藏、人格表达抢占了功能入口的注意力。

### 2. 交互反馈不够明确
- 某些按钮点击后反馈弱，用户会误以为“没反应”。
- 展开/收起、切页、继续引导、进入工坊这类动作缺少一致的视觉反馈。
- 默认状态、空状态、处理中状态、失败状态表达不统一。

### 3. 功能闭环不够短
- 环境、坐标配置、试运行、正式执行之间仍然有理解成本。
- 用户并不总能一眼看出“现在到底缺什么”。
- 工坊和战报虽然功能多，但主路径还不够短。

### 4. 全局可用性仍有风险
- 滚轮滚动、窗口高度适配、首屏信息截断仍是高优问题。
- 无边框窗口下的滚动与焦点管理需要持续兜底。
- 打包版要优先保证“可用”，再谈丰富体验。

### 5. 文案与产品语义还不够统一
- 一部分文案仍然偏抽象、偏系统术语。
- “陪刷 / 战报 / 工坊 / 桌宠详情 / 引导”之间的语言风格还没有完全收齐。
- 需要把所有关键提示都收成“人话”。
- 现有界面层还混入了一批编码污染和乱码字符串，会直接影响状态理解和错误判断。

### 6. 桌宠体验和工具体验还没有完全分层
- 桌宠壳应该负责常驻、陪伴、状态提醒。
- 页面主体应该负责执行具体任务。
- 现在两者仍有一定混杂。

## 实施原则

### 先稳再美
- 先解决滚动、点击、切页、反馈、白屏、打包可用性。
- 再继续做桌宠人格、收藏、成长演出。

### 先主路径再细节
- 先保证用户能顺利完成：
  1. 打开桌宠
  2. 看懂现在该做什么
  3. 开刷
  4. 记掉落
  5. 进工坊试运行

### 先收口再扩展
- 先把首页、战报、工坊都压成短路径。
- 再决定哪些信息适合折叠、哪些适合做成二级详情。

## 分步实施

### 第 1 步：全局可用性基线
目标：
- 保证每个页面都能稳定滚动、切页、点击、展开收起。

要做：
- 统一滚动容器与滚轮行为。
- 补强按钮点击反馈、切页反馈、空态/忙碌态/失败态。
- 给异常渲染加兜底，避免工坊或战报再次白屏。

验收标准：
- 三个主页面都能完整滚到底。
- 点击导航和主按钮都有明确反馈。
- 没有“点了没反应但其实状态变了”的情况。

### 第 2 步：首页压缩成任务仪表盘
目标：
- 首页只保留“现在做什么、上次做到哪、怎么开始”。

要做：
- 继续压缩桌宠壳默认高度。
- 首页默认只保留主动作、继续点、开跑表单。
- 统计、地图分布、战果预览全部折进“今日详细面板”。

验收标准：
- 用户第一屏就能知道当前主动作。
- 首页无需滚动也能看到核心按钮。

### 第 3 步：工坊闭环重构
目标：
- 让工坊变成“补条件 -> 录配置 -> 试运行 -> 看结果”的单路径页面。

要做：
- 只保留当前主任务和当前选中任务卡。
- 环境修复站、详细预检、历史日志全部二级展开。
- 明确显示“缺环境 / 缺坐标配置 / 可试运行 / 可正式执行”。

验收标准：
- 用户进入工坊后 5 秒内能知道自己下一步该点什么。
- 录制、试运行、看日志不会迷路。

### 第 4 步：战报闭环重构
目标：
- 让战报变成“快速录入 -> 最近几条 -> 详细筛选/导出”的顺序。

要做：
- 默认只保留快速录入和最近记录。
- 详细筛选、周报、导出、完整明细全部折叠。
- 强化 Ctrl+V 贴图后的即时反馈和保存成功反馈。

验收标准：
- 用户第一次打开战报就知道如何记第一条掉落。
- 不展开详细区也能完成最常用的记录动作。

### 第 5 步：桌宠壳与详情分层
目标：
- 桌宠默认是“轻壳”，详情才承载成长、收藏、设定。

要做：
- 默认头部只保留状态、悬浮/置顶/收起、一个详情入口。
- 成长、房间、收藏、系统开关全部放进“桌宠详情”。
- 保证桌宠不压缩功能页面的可视区域。

验收标准：
- 默认桌宠头部显著变短。
- 主要页面可视区域明显增大。

### 第 6 步：文案统一与错误解释
目标：
- 所有关键提示都能让第一次使用的人看懂。

要做：
- 收齐首页、工坊、战报、桌宠壳文案。
- 批量清理界面里的乱码和编码污染字符串。
- 去掉抽象表达和过度世界观表达。
- 把错误提示统一翻译成人话。

验收标准：
- 不看代码、不看日志，也能理解大部分关键状态。

### 第 7 步：桌宠个性化体验回填
目标：
- 在不干扰主功能的前提下恢复陪伴感。

要做：
- 把随机对白、钓鱼、成长、收藏演出放到二级层。
- 让这些表达服务主流程，而不是抢主流程。

验收标准：
- 桌宠依然有角色感，但不再遮挡任务路径。

### 第 8 步：发布前回归
目标：
- 做一轮真实可用性收口。

要做：
- 打开桌宠 -> 开刷 -> 记掉落 -> 进工坊 -> 试运行 这条完整链路回归。
- 验证滚动、点击、托盘、悬浮、打包版路径。
- 收尾残余白屏、乱码、布局溢出问题。

验收标准：
- 打包版可以作为主使用版本连续使用。

## 建议执行顺序

1. 第 1 步：全局可用性基线
2. 第 2 步：首页压缩成任务仪表盘
3. 第 3 步：工坊闭环重构
4. 第 4 步：战报闭环重构
5. 第 5 步：桌宠壳与详情分层
6. 第 6 步：文案统一与错误解释
7. 第 7 步：桌宠个性化体验回填
8. 第 8 步：发布前回归

## 当前下一步

下一轮直接从 `第 1 步：全局可用性基线` 开始，先统一滚动、点击反馈、切页反馈和空态/失败态。
