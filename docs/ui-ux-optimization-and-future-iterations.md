# 暗黑2桌宠 - UI/UX 优化与未来迭代方向技术文档

## 1. 当前软件整体分析与设计缺陷

### 1.1 核心定位与现状
目前软件定位于“暗黑2玩家的桌面宠物与刷图辅助工具”，集成了“陪刷（记录与状态）”、“战报（掉落识别与管理）”、“工坊（自动化辅助）”三大核心模块。
整体架构使用了 React + TypeScript + Electron，具备较好的跨平台和系统底层交互能力。

### 1.2 UI/UX 设计缺陷 (针对大规模玩家使用场景)
1. **引导流程存在认知断层**：
   - 玩家初次上手时，信息密度过大（引导、战报、工坊、桌宠概念混杂），没有一条极简的“Happy Path”（如：一键开启 -> 开始记录 -> 查看掉落）。
   - 桌宠状态（悬浮窗模式）与主面板模式（Panel）的切换逻辑不够直观，玩家容易找不到核心功能入口。
2. **面板层级与信息展示不够清晰**：
   - `CounterPanel` 与 `PetShell` 之间的状态控制存在耦合（如之前发生的 `showCompanionDetails` 导致主体内容消失的 Bug），扩展信息与核心操作未能有效分离。
   - 战报模块缺乏高价值掉落的视觉强反馈，普通的掉落列表容易让玩家产生疲劳。
3. **自动化与工坊操作门槛高**：
   - 工坊环境依赖 Python 及多项配置，尽管有预检（Preflight）机制，但失败后的报错提示（如依赖缺失、环境错误）对非技术玩家不够友好。
4. **容错与异常处理体验生硬**：
   - 虽然引入了 `PanelErrorBoundary`，但在极端情况下（如组件崩溃），恢复手段只有“重置页面”，缺乏平滑的降级体验。
5. **多开与游戏启动集成不足**：
   - 玩家仍需依赖外部工具（如 handle64 或其他批处理脚本）来解除游戏互斥体实现多开，启动流程与桌宠本身割裂，未能形成一站式服务。

## 2. 后续迭代功能方向与优化方案

### 2.1 交互体验 (UX) 升级
- **极简模式与深度模式分离**：为新手提供极简的“仅计次+掉落记录”模式，隐藏工坊与复杂环境配置；为硬核玩家提供深度模式。
- **智能化场景感知**：根据玩家当前运行的游戏画面（如通过图像识别），自动切换“陪刷”地图路线，无需手动点击。
- **桌面级沉浸体验优化**：优化悬浮宠物的动画、拖拽手感、以及对多显示器和游戏全屏/无边框窗口模式的兼容性。
- **一站式启动与多开管理**：内置基于 Rust 的轻量级启动器，玩家可在桌宠面板中一键检测路径、清理互斥体并自动登录游戏，无需繁琐的外部操作。

### 2.2 界面视觉 (UI) 升级
- **暗黑破坏神主题深度契合**：优化字体、边框、按钮材质（如添加类似暗黑2风格的金属质感与羊皮纸底纹），提升沉浸感。
- **掉落高光特效**：引入暗金、符文、符文之语等特殊掉落的专属动画提示，并允许一键生成带游戏截图的“炫耀卡片”供社区分享。

## 3. 技术执行拆分文档 (分布执行、验证、验收、测试)

为了将上述方向落地，现拆分为以下几个阶段的技术任务，以便逐步推进。

### 阶段一：核心 UI 组件解耦与稳定性修复 (已完成)
**目标**：解决面板状态耦合导致的内容丢失问题，提升基础交互的健壮性。
- **执行**：
  1. 重构 `App.tsx` 中的条件渲染逻辑，将 `CounterPanel`（核心主控）的可见性与 `showCompanionDetails`（扩展信息）完全解耦。
  2. 修复 `main.css` 中布局容器可能导致的 overflow 隐藏问题。
- **验证**：检查在不同模式下（极简、展开扩展）主功能按钮是否始终可用。
- **验收**：玩家启动后能稳定看到“陪刷首页”和计次按钮。
- **测试**：在开发与打包环境中，反复切换“展开/收起扩展状态”，观察页面渲染树无异常抛出。

### 阶段二：新手指引与环境配置自动化降级 (已完成)
**目标**：降低普通玩家使用“工坊”和“自动化”的门槛。
- **执行**：
  1. 优化 `SetupGuide` 组件，加入一键静默安装 Python 运行环境及依赖的流程（通过 `handleInstallAllEnvironments` 结合）。
  2. 封装环境错误提示，将底层堆栈报错转化为白话文（如：“未检测到游戏运行”、“权限不足请以管理员运行”）。
- **验证**：在未安装 Python 的干净 Windows 虚拟机中运行工具。
- **验收**：点击一键配置后，环境能够自动就绪并完成 Preflight 预检。
- **测试**：覆盖无网络、权限不足、安装包损坏等异常场景的重试机制。

### 阶段二点五：快速启动器与多开功能整合 (已完成)
**目标**：将独立的 D2R 快速启动器（基于 Rust）深度集成到桌宠 UI 中，提供一站式游戏启动体验。
- **执行**：
  1. 分析并重构 D2R-fast-launcher 源码，将其核心逻辑（注册表读取、进程互斥体清理、启动与自动输入）编译为独立的 CLI 可执行文件 (`fast-launcher.exe`)。
  2. 在 Electron 主进程中添加 `fast-launcher:xxx` IPC 接口，通过 `execFile` 与 Rust 可执行文件进行 JSON 格式的稳定通信。
  3. 在前端 React 层扩展 Types，并在 `CounterPanel` 中引入全新的 `FastLauncherCard` 组件，提供路径自动检测、多开清理、带账号密码的快捷启动表单。
- **验证**：在本地环境运行，点击“自动检测”能否正确读取注册表中的游戏路径；点击“清理互斥体”能否正确关闭 D2R.exe 的多开限制句柄。
- **验收**：玩家能够在桌宠的“陪刷”主界面展开快速启动器面板，一键完成多开准备和游戏启动。
- **测试**：测试错误边界（如未安装游戏、路径包含空格或特殊字符、账号密码传递参数时的转义处理）。

### 阶段三：视觉主题重构与掉落反馈增强 (已完成)
**目标**：全面提升 UI 质感与“爆装”的情绪价值。
- **执行**：
  1. 提取一套独立的主题 CSS Variables，替换当前较简单的配色方案。
  2. 扩展 `DropPanel`，针对 `data.drops` 中价值评分高的物品（如 24 号以上符文）注入粒子发光特效（CSS 动画 + Canvas 辅助）。
- **验证**：导入模拟的暗金掉落数据，查看渲染表现。
- **验收**：特效流畅不掉帧，且分享截图功能排版精美。
- **测试**：性能测试，确保在后台记录大量掉落时内存占用不会出现泄露。

### 阶段四：悬浮窗与游戏深度融合 (待执行)
**目标**：将工具箱打造成无感知的游戏内伴侣。
- **执行**：
  1. 优化 Electron 的 `BrowserWindow` 属性（透明度、鼠标穿透、置顶策略）。
  2. 开发通过 OCR 识别左上角地图名自动切换 `activeRun` 的核心引擎。
- **验证**：与暗黑2重制版同时运行，测试置顶覆盖与识别准确率。
- **验收**：无需玩家切出游戏即可自动记录刷图轮次和耗时。
- **测试**：长时间挂机测试，评估 OCR 模块对 CPU 的消耗。
