# 暗黑 2 桌宠助手功能基线

## 1. 文档目的

这份文档用于冻结当前版本的产品边界，作为后续 `Sprint A` 到 `v1.0` 的回归基线。

它回答三个问题：

1. 当前版本到底已经有什么能力。
2. 这些能力分别出现在哪些页面和模式里。
3. 后续重构 UI / 交互时，哪些能力不能被不小心改丢。

---

## 2. 当前信息架构

## 桌宠模式

当前存在两种窗口模式：

- `panel`
  也就是主面板模式，承载完整导航和三大一级页面。
- `floating`
  也就是悬浮桌宠模式，承载宠物壳、状态气泡和快捷入口。

## 一级页面

主面板模式下包含三个一级页面：

1. `陪刷`
2. `战报`
3. `工坊`

## 系统级常驻能力

无论是否在主面板页面，系统级能力都存在：

- 托盘
- 全局快捷键 `Alt+Shift+D`
- 收起到托盘
- 始终置顶
- 开机自启
- 系统通知

---

## 3. 页面职责基线

## 3.1 陪刷

### 目标

承接高频刷图主流程，负责“开始 / 结束 / 统计 / 快捷跳转”。

### 当前能力

- 开始一次刷图记录
- 结束一次刷图记录
- 展示本次进行中耗时
- 展示今日次数 / 总耗时 / 平均单次
- 展示最近刷图列表
- 展示今日掉落预览
- 跳转到战报
- 跳转到工坊

### 不可回退项

- 必须允许在首页直接开始和结束刷图
- 今日统计必须在首页一眼可见
- 最近战果必须能从首页快速跳转

---

## 3.2 战报

### 目标

承接刷图收口动作，负责“截图 / OCR / 记录 / 筛选 / 回看”。

### 当前能力

- `Ctrl+V` 粘贴截图
- 截图保存到本地
- 调起 OCR
- 回填物品名建议
- 回填备注建议
- 保存掉落记录
- 展示今日战报摘要
- 按关键词 / 类别 / 场景 / 高亮筛选
- 查看截图文件

### 不可回退项

- 粘贴截图必须仍然可用
- OCR 失败不能阻塞掉落记录保存
- 战报列表必须能回看今日记录

---

## 3.3 工坊

### 目标

承接低频但高价值的自动化动作和 profile 运维。

### 当前能力

#### 符文

- 输入数量
- 试运行
- 立即执行
- 查看 profile
- 录制 profile
- 导入旧配置
- 查看日志

#### 宝石

- 矩阵输入
- 截图识别输入
- `Ctrl+V` 粘贴宝石截图
- 试运行
- 立即执行
- 查看 profile
- 录制 profile
- 导入旧配置
- 查看日志

#### 金币

- 输入总额
- 输入角色等级
- 试运行
- 立即执行
- 查看 profile
- 录制 profile
- 查看日志

### 不可回退项

- 三类任务都必须保留 `试运行 / 立即执行`
- 日志查看能力不能移除
- profile 路径和录制入口必须仍可达

---

## 4. 悬浮态职责基线

悬浮态不是小号工坊，也不是缩小版完整面板。

### 当前职责

- 展示桌宠状态
- 展示今日刷图 / 掉落摘要
- 展示高亮掉落气泡
- 快速展开主面板
- 快速跳到战报
- 快速跳到工坊
- 快速切换置顶
- 快速收起

### 不可回退项

- 悬浮态必须比面板态轻量
- 至少保留一个“展开主面板”主按钮
- 必须保留对战报和工坊的快速入口

---

## 5. 数据基线

当前数据核心在 `AppData`。

### 核心字段

- `activeRun`
- `runHistory`
- `drops`
- `integrations`
- `automationDrafts`
- `settings`

### 本地目录

典型目录位于：

`C:\Users\<user>\AppData\Roaming\d2-desktop-pet\d2-pet\`

### 当前落盘资产

- `data.json`
- `screenshots\YYYY-MM-DD\`
- `automation-logs\YYYY-MM-DD\`

### 不可回退项

- 修改 UI 时不能破坏现有 `data.json` 兼容性
- 掉落截图和自动化日志目录结构不能随意改变

---

## 6. 关键状态基线

## 运行状态

- 启动中
- 空闲
- 刷图进行中
- 刚记录高亮掉落
- 自动化运行中
- 自动化运行完成
- 自动化运行失败

## 页面状态

每个页面至少需要覆盖：

- 正常态
- 空态
- 忙碌态
- 错误态

## 提示反馈

当前主要通过以下方式反馈：

- 页内 toast
- 状态 pill / chip
- 气泡提示
- 系统通知

## 7. 高频主流程基线

## 流程 A：开始并结束一次刷图

1. 在陪刷页输入地图名
2. 点击开始
3. 计时开始
4. 点击结束
5. 记录写入历史和今日统计

## 流程 B：记录一次掉落

1. 切到战报
2. 粘贴截图
3. 等待 OCR
4. 检查回填结果
5. 保存记录

## 流程 C：运行工坊任务

1. 切到工坊
2. 选择任务
3. 填输入参数
4. 先试运行
5. 查看日志
6. 再正式执行

## 流程 D：桌面常驻

1. 启动桌宠
2. 收起到托盘
3. 用全局快捷键呼出
4. 切换悬浮态 / 面板态

---

## 8. 目前已知薄弱点

这些点不是当前版本缺失，而是后续 Sprint A / Sprint B 优先要打磨的地方：

- 文案和状态反馈还不够统一
- 失败提示还偏技术口径
- OCR 纠错层还不够强
- 工坊日志目前偏“原始输出”，缺少摘要
- UI 视觉虽已成型，但设计系统尚未文档化

---

## 9. 使用方式

后续每做一批改动，都应该拿这份文档对照三件事：

1. 有没有改动一级页面职责。
2. 有没有破坏四条高频主流程。
3. 有没有把系统级常驻能力改丢。
